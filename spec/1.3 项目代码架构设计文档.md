# 1.3 项目代码架构设计文档（工具小站）

> 依据 `spec/1.1功能描述.md` 与 `spec/1.2网站示意图.txt` 进行的简要代码架构设计与开发计划。

---

## 一、是否需要后端服务的判断

当前功能列表：
- 文件转换
- 图片压缩
- 图片裁剪
- 图片格式转换（JPG/PNG/WebP 等）
- 图片水印（文字/图片）

**判断：**
- 以上功能都可以基于浏览器的 File API + Canvas 等能力，通过前端在本地完成处理（不上传服务器），优点是：
  - 实现简单、部署方便（静态站即可上线）。
  - 用户隐私更好（文件不出本地）。
- 如果后续需要：
  - 支持更复杂的文档格式转换（如 Office、PDF 深度处理）；
  - 做批量任务队列、用户登录历史记录等，
  才更有必要引入后端。

> **结论：本版本代码架构设计采用「纯前端实现，无后端服务」的方案。**  
> 后端服务可以在未来迭代中按需补充，此处不展开设计。

## 二、技术选型（前端）

- **基础技术栈**：HTML + CSS + 原生 JavaScript
- **页面组织**：多页面结构（入口首页 + 各工具独立页面），方便初学者理解与维护。
- **图片处理**：
  - 使用 `<canvas>` + JavaScript 完成压缩、裁剪、格式转换与水印叠加。
- **文件转换**：
  - 本版仅支持简单的「图片类文件格式转换」（例如 PNG ↔ JPG ↔ WebP），与“图片格式转换”工具统一实现。
  - 更复杂的文档转换（如 Word/PDF）暂不实现，只在 UI 上进行范围提示。

## 三、功能模块与页面划分

对应 `1.2网站示意图` 中的首页结构：

1. **首页（index.html）**
   - HEADER 顶部导航：LOGO / 首页 / 工具 / 帮助 / 关于
   - HERO 主视觉：标题「在线文件与图片小工具，一站式高效处理」+ CTA 按钮
   - TOOLS 区：卡片入口（文件转换 / 图片压缩 / 图片裁剪 / 图片格式转换 / 图片水印）
   - GUIDE 区：使用三步说明
   - FOOTER：版权及链接

2. **工具页面**
   - `tools/file-convert.html`：文件转换（当前版本只做图片类格式转换或给出「暂不支持」说明）。
   - `tools/image-compress.html`：图片压缩。
   - `tools/image-crop.html`：图片裁剪。
   - `tools/image-format.html`：图片格式转换。
   - `tools/image-watermark.html`：图片水印。

3. **脚本与样式**
   - `assets/styles.css`：全站样式。
   - `assets/main.js`：首页通用交互（导航高亮、滚动到「常用小工具」等）。
   - `assets/tools/fileConvert.js`：文件转换工具逻辑。
   - `assets/tools/imageCompress.js`：图片压缩逻辑。
   - `assets/tools/imageCrop.js`：图片裁剪逻辑。
   - `assets/tools/imageFormat.js`：图片格式转换逻辑。
   - `assets/tools/imageWatermark.js`：图片水印逻辑。

## 五、数据流与基本交互（前端）

以「图片压缩」为例说明，其它图片类工具类似：

1. 用户在 `image-compress.html` 点击上传按钮或拖拽图片文件。
2. 前端读取文件（`FileReader`），在 `<img>` 或 `<canvas>` 中展示预览。
3. 用户调整压缩参数（质量、目标宽度等）。
4. 点击「开始压缩」按钮：
   - 在前端使用 `canvas.toBlob` / `toDataURL` 生成压缩后的图片。
5. 前端提供「下载结果」按钮，触发浏览器下载，无需后端参与。

文件转换 / 裁剪 / 格式转换 / 水印工具的差异仅在于：
- 读取文件后，对应的 Canvas 操作不同；
- 参数表单不同（如水印文字、位置、透明度等）。

## 六、开发计划（简化版）

> 目标：用尽量少的步骤完成一个可用版本，后续再逐步优化。

**阶段 1：项目骨架与基础页面（1~2 天）**
- 创建 `site-tools/` 目录与基础 HTML 结构（首页 + 空白工具页面）。
- 编写 `assets/styles.css`，复用 `1.2网站示意图` 的布局与视觉层级。
- 实现顶部导航与「常用小工具」卡片跳转逻辑（`assets/main.js`）。

**阶段 2：核心图片工具功能（3~4 天）**
- `image-compress.html` + `imageCompress.js`：完成图片压缩流程（上传 → 预览 → 压缩 → 下载）。
- `image-crop.html` + `imageCrop.js`：允许用户选择裁剪区域并导出裁剪图。
- `image-format.html` + `imageFormat.js`：在前端完成 PNG/JPG/WebP 之间的转换。
- `image-watermark.html` + `imageWatermark.js`：支持文字水印（位置、大小、透明度），必要时再加图片水印。

**阶段 3：文件转换与使用说明（1~2 天）**
- `file-convert.html` + `fileConvert.js`：
  - 当前版本仅支持图片类文件格式转换，与 `image-format` 逻辑共用或复用。
  - 对「非图片类型」给出提示「当前版本暂不支持该格式」。
- 完善首页「如何使用」区的链接跳转与说明一致性。

**阶段 4：体验优化与文档对齐（1 天）**
- 检查所有页面的文案与入口是否与 `1.1功能描述`、`1.2网站示意图` 一致。
- 补充必要的错误提示（如未选择文件就点击开始处理）。
- 更新 `README.md` 或 `spec` 中的使用说明（如新增本工具站的预览方式）。
