# UI 原型与页面约定

参考若依默认「系统管理 → 部门管理」类页面，**用户管理**与**部门管理**保持同一套交互范式，便于用户学习与维护。

## 布局

- **顶栏**：系统名称、用户信息、布局设置、退出等（与若依默认一致即可）。
- **侧栏**：多级菜单；**系统管理**下包含 **用户管理**、**角色管理**、**菜单管理**、**部门管理**等。
- **内容区**：面包屑 + **多页签（Tab）** + 当前页面。

## 列表页（与用户管理对齐）

1. **查询区**
   - 输入框：如用户名称、手机号等（按业务字段）。
   - 下拉：状态（正常/停用）等。
   - 按钮：**搜索**、**重置**。

2. **工具栏**
   - **新增**（主色按钮）。
   - 可选：导入、导出、批量删除等（若依常见模式）。

3. **数据表格**
   - 列：与数据库用户表及业务展示字段一致（如用户编号、登录名、昵称、部门、状态、创建时间）。
   - **操作**列：**修改**、**删除**、重置密码等（权限控制由若依 `v-hasPermi` 等指令实现）。

4. **分页**
   - 底部分页组件，与若依 `pagination` 封装一致。

## 部门管理（树表）与用户管理（列表）

- **部门管理**：多为左树右表或树形表格，操作列含「新增子部门」等。
- **用户管理**：一般为普通表格 + 弹窗表单；若需要「按部门筛选」，可在查询区增加部门树选择器。

## 设计稿与资源

若依自带主题与组件即可高度还原原型；如需定制主题色，在 Element Plus 变量或若依主题配置中调整。
